# 成本效率：TAT 块架构

TAT 基于块的存储相比原始 markdown/JSON/CSV 日志减少 **25–30 倍** 的内存占用和令牌使用，同时通过连接优先于特征的结构优先级 (`connection_weight = 4`) 保留 100% 有意义的信息。

## 真实场景：具有持久记忆的代理系统

**系统：** 一个典型的基于 LLM 的代理，具有跨多个月操作的持久会话记忆。
**TAT 之前的格式：** Markdown 会话日志、CSV 评估追踪、JSON 因果索引、定期健康报告。

### TAT 之前（原始存储）

| 指标 | 值 |
|------|-----|
| 每月会话数 | ~600 |
| 每个会话的数据 | ~10 KB (markdown + CSV) |
| 总存储 (4 个月) | ~24 MB |
| 总令牌数 | ~1200 万 |
| 每月 LLM 分析成本* | ~$24.00 |

*假设 100% 会话数据通过 LLM 分析，价格为 $2/100 万令牌。

### TAT 之后（块架构）

| 指标 | 值 |
|------|-----|
| 每个会话的数据 (压缩后) | ~0.33 KB |
| 总存储 (4 个月) | **<1 MB** |
| 总令牌数 | **~400,000** |
| 每月 LLM 分析成本* | **~$0.80** |

**节省：存储减少 30 倍，LLM 成本减少 30 倍。**

### 额外节省：块轮播

块轮播仅将 **热块** (高因果密度) 传递给 LLM：

- LLM 仅分析约 20% 的数据
- 在压缩基础上额外 **降低 5 倍成本**
- 同一系统的 **实际每月成本：~$0.16**

## 节省背后的架构

1. **五层块** (Theme, Role, Emotion, Meaning, Goal) — 分离特征和连接
2. **四种密度格式** (pkl, bin, json, txt) — 仅需核心 (bin)
3. **镜像时间戳** — 直接机器访问，零解析开销
4. **连接优先** — 56% 特征丢失无质量下降

## 适用性

任何具有会话数据的系统：
- 代理身份平台
- 客户支持日志
- 持续学习基准
- 长期运行的 LLM 应用
